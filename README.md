# 사주 대소사 예측 프롬프트 생성기

생년월일시를 입력하면 **Claude AI 전용 정밀 사주 분석 프롬프트**를 자동으로 생성합니다.

## 기능

- 생년월일시 + 성별 입력
- 사주팔자 자동 산출 (연주·월주·일주·시주)
- 오행 강약·십성·충형합 자동 계산 및 프롬프트 반영
- 대운·세운 자동 배열
- 인생 대소사 15개 도메인 선택 분석
  - 결혼·동거·재혼
  - 이혼·별거·이별
  - 연애·이성운
  - 출산·자녀운
  - 취업·승진·이직
  - 사업성패·창업
  - 재물획득·손실·투기
  - 건강이상·수술·질병
  - 사고·부상·재난
  - 소송·관재·법적문제
  - 이사·이민·해외운
  - 학업·시험·자격증
  - 명예·성공·몰락
  - 부모사망·형제갈등·가족운
  - 사망·생사 관련 극한운

## 사용법

1. `index.html`을 브라우저로 열거나 GitHub Pages로 배포
2. 생년월일시 입력
3. 분석받을 영역 선택
4. **프롬프트 생성하기** 클릭
5. **복사하기** → Claude.ai 대화창에 붙여넣기

## 음력(陰曆) 입력 지원

- 입력 화면에서 **양력 / 음력**을 라디오 버튼으로 선택할 수 있습니다.
- 음력 선택 시 **윤달(閏月) 체크박스**가 나타나며, 윤달 출생자는 반드시 체크해야 정확히 변환됩니다.
- 변환에는 `js/data/manseryeok.js`(KASI 한국천문연구원 음양력변환계산 데이터 기반, 1900~2050년 지원)를 사용합니다. 출처: [urstory/manseryeok-js](https://github.com/urstory/manseryeok-js) (MIT License)
- 음력으로 입력하면 화면과 생성된 프롬프트 양쪽에 "음력 OOOO년 OO월 OO일 → 양력 OOOO년 OO월 OO일로 변환되었습니다"라는 안내가 함께 표시됩니다.
- 변환 범위(1900~2050년)를 벗어나면 오류 메시지를 표시하고 프롬프트를 생성하지 않습니다.

## 베이스레이트 검증기 (탭 2)

프롬프트가 만든 해석이 "맞아서 적중한 것"인지, "어느 해를 찍어도 맞는 헐거운 규칙이라 우연히 맞은 것"인지 구분하기 위한 도구입니다.

### 배경

합·충·형·해·삼합·십성 등 사주명리의 신호는 종류가 많아서, 아무 연도나 짚어도 그중 하나는 거의 항상 걸립니다. "사건이 일어난 해에 무슨 신호가 있었지?"를 사후에 찾아 설명하는 방식은 이 문제 때문에 검증력이 없습니다 — 실제로 이 검증기로 확인한 한 사례에서는 "삼합(반합 포함)" 신호가 54년 전체에서 100% 발생해 변별력이 0이었습니다.

### 사용법

1. **베이스레이트 검증기** 탭으로 전환
2. 생년월일시(양력/음력 모두 지원)·성별 입력
3. 분석 종료 연도(보통 현재)와 **실제 사건이 있었던 연도**를 쉼표로 입력 (예: `1995, 1997, 2003, 2013`)
4. 삼합 인정 기준(완성만 / 반합도 인정) 선택
5. **베이스레이트 계산하기** 클릭

### 출력 읽는 법

- **변별력(%p)** = 사건 해 히트율 − 무사건 해 히트율. 0%p에 가까우면 그 신호는 사건을 구분하지 못한다는 뜻이고, 음수면 오히려 무사건 해에서 더 자주 뜨는 역효과입니다.
- 임계값(신호 1개/2개/3개 이상)별로 동시에 비교해, 어느 기준에서 변별력이 생기는지 직접 확인할 수 있습니다.
- 시주는 출생시각을 모르는 사람과도 공정하게 비교하기 위해 계산에서 제외하고, 연주·월주·일주(3기둥)만 사용합니다.
- 사건 표본이 적을 때(예: 4~5건)는 화면에 경고가 표시되며, 수치는 경향성 참고용으로만 활용해야 합니다.

### 설계 원칙

- 신호별 가중치·합산 점수를 미리 매기지 않습니다 — 점수화 자체가 또 다른 사후 끼워맞추기가 될 수 있기 때문입니다.
- 분석 규칙은 `js/baserate.js` 상단에 버전(`BASERATE_RULE_VERSION`)으로 고정해두고, 사례를 볼 때마다 임의로 바꾸지 않는 것을 원칙으로 합니다. 규칙을 바꿀 필요가 있다면 버전을 명시적으로 올리고 무엇을 바꿨는지 기록해야 합니다.

## 구조

```
saju-prompt-generator/
├── index.html          # 메인 UI (탭 2개: 프롬프트 생성기 / 베이스레이트 검증기)
├── css/
│   └── style.css       # 스타일
└── js/
    ├── data/
    │   ├── manse_100_db.js     # 100년 만세력 DB (1950~2050) — 연주/일주 산출용
    │   ├── manse_100_db.json   # 위와 동일 데이터(JSON 원본, 참고용)
    │   ├── solar_terms_db.js   # 절기 DB (2010~2050) — 월주/대운 정밀 산출용
    │   └── manseryeok.js       # 음력↔양력 변환 DB (1900~2050) — 음력 입력 지원용
    ├── saju.js         # 사주 계산 엔진 (천간·지지·십성·충형합·대운·세운)
    ├── prompt.js       # 프롬프트 생성 엔진
    ├── main.js          # 프롬프트 생성기 탭 UI 이벤트 핸들러 (양력/음력 토글 포함)
    ├── baserate.js      # 베이스레이트 계산 엔진 (신호 판정·임계값별 변별력 계산)
    ├── baserate-ui.js   # 베이스레이트 탭 UI 이벤트 핸들러·결과 렌더링
    └── tabs.js          # 탭 전환 로직
```

### 산출 우선순위 (정밀도 순)

| 기둥 | 1순위 (정밀) | 2순위 (근사 폴백) |
|---|---|---|
| 연주 | 절기 DB로 입춘 날짜 확정 + 만세력 DB `year_ganji` 조회 | 60갑자 순환 공식 |
| 월주 | 절기 DB로 정확한 절기 경계 탐색 + 오호둔법(五虎遁法) | 월 숫자 기준 근사 |
| 일주 | 만세력 DB `daily_cache` 직접 조회 (1950~2027) | `jan_1st_ganji` + 경과일수 환산 (~2050) |
| 대운 시작 나이 | 절기까지 실제 일수 ÷ 3 (전통 공식) | 월중 날짜 기준 근사 (3~9세) |

- 절기 DB 범위(2010~2050)를 벗어나는 생년은 월주·대운이 **근사치**로 계산되며, 생성된 프롬프트 상단에 `⚠` 표시로 자동 안내됩니다.
- 절기 DB의 `ganji_year` 텍스트 필드는 연도별 표기 기준이 일관되지 않아 신뢰도가 낮아 사용하지 않으며, 입춘 **날짜**만 경계값으로 사용하고 실제 간지는 만세력 DB로 산출합니다.

## 분석 구조 (프롬프트 내 포함 항목)

### FOUNDATION LAYER (원국 분석)
- 오행 강약 분포
- 일간 강약 (신강/신약)
- 용신·희신·기신·구신·한신 확정
- 십성 배치 및 육친 분석
- 격국 확정
- 형·충·합·파·해·공망·역마·도화 분석

### TIMELINE LAYER (운세 흐름)
- 대운 (10년 주기)
- 세운 (1년)
- 월운 (월 단위 정밀화)

### 용신 발동 사이클
- 활성 / 약화 / 무력화 / 재발동 구간 추적

### EVENT DOMAIN (인생 대소사)
- 도메인별 십성·충형합·용신 발동 기준 완전 적용

## 주의사항

- 절기 DB는 2010~2050년만 커버합니다. 이 범위를 벗어나는 생년월일은 월주·대운이 근사치로 계산되며, 생성된 프롬프트에 자동으로 ⚠ 표시됩니다.
- 만세력 DB는 1950~2050년을 커버합니다. 이 범위를 벗어나면 60갑자 순환 공식으로 폴백합니다.
- 음력 변환 DB(manseryeok.js)는 1900~2050년만 커버합니다. 이 범위를 벗어나는 음력 날짜는 변환할 수 없으며, 오류 메시지가 표시됩니다.
- 생성된 프롬프트는 Claude.ai (claude.ai)에서 사용하도록 설계되었습니다.
